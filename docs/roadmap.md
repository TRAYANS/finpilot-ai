# FinPilot AI - Roadmap inicial

## Fase 0 - Definicion

Objetivo: convertir la idea en una especificacion clara.

Entregables:

- Vision del producto.
- Mapa de pantallas.
- Direccion visual.
- Especificacion del Dashboard.
- Modelo de datos inicial.
- Reglas del Coach IA.

## Fase 1 - Prototipo visual

Objetivo: disenar la primera version navegable del Dashboard.

Entregables:

- Wireframe del Dashboard.
- Mockup visual desktop.
- Mockup visual mobile.
- Estados vacios.
- Estados con datos.
- Panel de salud financiera.
- Tarjeta Coach Financiero.

## Fase 2 - Prototipo funcional local

Objetivo: crear una app inicial navegable con datos ficticios.

Stack recomendado:

- Next.js
- TypeScript
- Tailwind CSS
- Recharts o Tremor para graficas
- Supabase como backend futuro

Pantallas:

- Dashboard
- Patrimonio
- Coach IA
- Objetivos

## Fase 3 - Modelo de datos y persistencia

Objetivo: guardar datos reales manuales.

Funcionalidades:

- Crear cuentas.
- Crear activos.
- Crear pasivos.
- Registrar movimientos.
- Calcular patrimonio neto.
- Calcular salud financiera.

## Fase 4 - IA integrada

Objetivo: conectar el Coach IA con los datos del usuario.

Funcionalidades:

- Resumen mensual.
- Recomendaciones.
- Riesgos.
- Objetivos.
- Modo CEO.

## Fase 5 - Integraciones

Objetivo: automatizar datos.

Posibles integraciones:

- Open banking.
- Brokers.
- Cripto.
- Precios de mercado.
- Documentos.

## Proxima decision

La siguiente decision importante es elegir el entregable inmediato:

1. Wireframe del Dashboard.
2. Prototipo HTML/CSS del Dashboard.
3. Documento ampliado de producto.
4. Arquitectura tecnica de la app.
